# Описание
Я хочу попрактиковатсья в использовании Git, поэтому сделаю игру на Lua. Это будет текстовое приключение-квест про попытку пройти через подземную станцию метрополитена, имеющую крайне высокие потолки. Она в основном сделана из бетона, насколько я понимаю, в бруталистском стиле.

Ей стоит быть не особо большой, чтобы я её не успел забросить. Позже она будет конвертирована в исполняемый файл через `srlua`.

# Организация файлов
Есть главный файл `main.lua`, который будет инициализировать игру, вызывать осуществление и загрузку сохранений, проверять возможность действий. То есть он должен лишь руководить, отдавая задачи модулям.

Эти модули помещаются в папку `modules`. В папке `texts` располагаются тексты, которые будут обработаны и отабражены на экране. Стоит ещё подумать, где будут располагаться сохранения, но папка будет называться `saves`.

Сами файлы соответсвуют `kebab-case`.

# Организация кода

Все переменные и функции по умолчанию локальные, названия обычных используют `camelCase`, константы — `CONSTANT_CASE`, вспомогательные функции — `snake_case`, остальные функции — `PascalCase`.

Для указания типов переменных, принимаемых аргументов и возвращаемых значений, полей классов, содержимого таблиц используется `Lua Language Server`.

Каждая функция должа отвечать за отдельный набор задач.
